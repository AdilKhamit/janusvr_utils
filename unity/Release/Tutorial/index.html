Download the JanusVR Exporter

You can get the exporter by 1 of 2 ways:

Github Source

The JanusVR Unity Exporter is in constant development in the JanusVR Utils Github page: https://github.com/JamesMcCrae/janusvr_utils

You can find the exporter project source code in the folder "unity/Project/JanusExporter", or here

Importing The JanusVR Exporter from Git

1. Install git if you haven't

2. git clone https://github.com/JamesMcCrae/janusvr_utils

3. Copy the folder "unity/Project/JanusExporter/Assets/JanusExporter" to your project


Unity Package

Download the latest release of the JanusVR Exporter by going to the JanusVR Utils GitHub Releases page

Importing The JanusVR Exporter

1. Import the JanusVR Package by going on "Assets/Import Package/Custom Package"

2. Select the package you downloaded

If you already had the JanusExporter installed, make sure to completely delete the JanusExporter folder in your project before importing the latest version.


3. Open the exporter by hitting Window -> Janus VR Exporter


4. Dock the JanusVR Exporter window somewhere in your working area


Before Exporting

  • Lightmap Settings:
    • Baked Global Illumination: Checked
    • Lighting Mode: Subtractive (Unity 5.6+)
    • Lightmapper: Enlighten (Unity 5.6+)
  • Lightmap Optimization:
    • Realtime Global Illumination: Unchecked (faster lightmap build times as we cant export realtime lightmaps)

Documentation

Exporter Window Parameters

Asset Objects Description
Search Type The algorithm to use to extract the model files.
Uniform Scale A uniform scale to preprocess models.

Main Description
Export Path The folder were exporting the scene to
Use Euler Rotations If we should export the HTML using Euler rotation instead of using vector directions (only supported on Janus 59+)

Texture Description
Force ReExport If we should force exporting all textures to the desingated texture format.
Unsupported Textures Format If force reexport is checked all material textures will be exported using the specified format.
Textures Quality If the unsupported format is a lossy format, choose the quality here

Scene Description
Export Inactive Objects If we should export inactive game objects
Export Dynamic Objects If we should export objects that are set as dynamic (i.e. not static)
Export Textures If we should export textures
Export Materials If the exporter should output the materials (if disabled, lightmaps are still exported, so you can take a look at only lightmap data with a gray tone)
Export Skybox Exports the scene's skybox

Exporter Window Actions

Export HTML only

This button is intended for when you have already exported the entire scene, but made a change that doesnt impact the already exported models/textures (like changing the position of an object in the scene).

Reset Parameters

Resets all parameters to their default values.

Full Export

Exports the entire scene - models, textures and the html file - to the output directory.


Lightmap Options

Option Description
None No lightmaps are exported.
Packed Packed into multiple atlases.
Packed Source EXR The optimal type of lightmaps: everything is exported straight out from Unity, packed into multiple atlases with HDR support.
Unpacked Unpacks the atlases into smaller textures, so no objects share the same lightmap texture. Consumes more memory with no gain. (Exists only for compatibility/debugging purposes)
Baked Bakes the lightmap into the object material. Consumes too much memory while making the main material look poor. (Exists only for compatibility/debugging purposes)

Custom Components

Room Entry Portal

GameObject -> JanusVR -> Room Entry Portal

Makes an entry portal on your Unity scene. You can position the portal and move it around, and this is going to be the entrance to your JanusVR room.

Link

GameObject -> JanusVR -> Link

Makes a link portal on your Unity scene. You can set up the URL, title and all other options, and this is going to be exported as a Link portal to JanusVR.
